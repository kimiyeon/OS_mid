# 프로젝트 목표: 구조화된 멀티 에이전트 찬반 토론 시스템 구축 (1인 개발)

## 1. 프로젝트 개요

사용자가 제시한 논쟁적 주제에 대해, 여러 AI 에이전트가 각각의 명확한 역할(생성, 비판, 반박/수정, 합의)과 제약을 가지고 구조화된 찬반 토론을 수행하고, 최종적으로 합의된 결과를 도출하는 시스템을 개발합니다. 1인 개발 프로젝트로서, AI 에이전트를 적극적으로 활용하여 시스템의 핵심 로직 구현부터 인프라 구축, 테스트, 문서화, 그리고 코드 품질 개선까지 전반적인 개발 과정을 효율적으로 진행합니다.

## 2. 핵심 목표

-   사용자 입력에 따라 논쟁적 주제를 분석하고 토론 전략 수립 (`Plan Agent` 구현)
-   다양한 관점에서 주장을 생성, 비판, 반박/수정하는 실행 에이전트 구현 (`Execution Agents`: Generator, Critic, Defender/Refiner, Judge)
-   토론 과정 및 결과의 논리적 오류, 누락, 제약 위반 여부 검증 (`Verification Agent` 구현)
-   검증 결과에 따른 토론 재실행 또는 최종 결과 확정 (`Gate Agent` 구현)
-   모든 토론 과정 및 에이전트 활동 기록 (`Logging Agent` 구현)
-   Git 브랜치 전략 적용 및 Docker 컨테이너화
-   AI 에이전트를 활용한 코드 품질 및 문서화 개선

## 3. 개발자 (본인) 역할 및 AI 에이전트 활용 전략


<inlineAdSlot total="3" index="0" />### 3.1. 개발자 (본인) 역할



-   **주요 책임:** 프로젝트의 아키텍처 설계, 모든 에이전트의 핵심 로직 구현, 시스템 통합 및 테스트, 프로젝트 관리 및 의사 결정.
-   **세부 역할:**
    -   `Plan Agent`, `Execution Agents`, `Verification Agent`, `Gate Agent`, `Logging Agent` 등 모든 에이전트의 설계 및 구현.
    -   에이전트 간의 소통 프로토콜 및 데이터 흐름 정의.
    -   Dockerfile 작성 및 컨테이너화 전략 수립.
    -   `requirements.txt` 관리 및 의존성 고정.
    -   시스템 전반의 통합 테스트 및 기능 검증.
    -   `PROMPT.md`, `README.md`, `ai-usage-log.md`, `ralph-log.md` 등 모든 문서 관리 및 작성.
    -   Git 브랜치 전략 수립 및 커밋/PR 관리 (자기 자신에게 PR 발행 또는 직접 main에 병합).


<inlineAdSlot total="3" index="1" />### 3.2. AI 에이전트 활용 전략 (나의 코딩 조력자)



-   **주요 책임:** 코드 생성 보조, 버그 디버깅, 테스트 코드 작성, 문서화 지원, 코드 리팩토링 및 품질 개선 제안.
-   **세부 역할:**
    -   **초기 코드 생성:** 각 에이전트의 기본 구조, 특정 기능 구현을 위한 코드 스니펫 생성 요청 (예: "Generator Agent의 기본 프롬프트와 Flask 엔드포인트 코드 작성해 줘").
    -   **디버깅 및 오류 수정:** 발생한 에러 메시지를 제공하고 해결책 또는 수정된 코드 요청.
    -   **테스트 코드 작성:** 특정 함수 또는 에이전트의 유닛/통합 테스트 코드 생성 요청.
    -   **문서화 지원:** `README.md`의 특정 섹션 작성 보조, `ai-usage-log.md` 및 `ralph-log.md` 내용 정리.
    -   **리팩토링 및 품질 개선:** 작성된 코드의 효율성, 가독성, 최적화 방안에 대한 제안 요청. `Ralph Mode`를 통해 반복적으로 코드 품질 개선 시도.
    -   **아이디어 확장 및 대안 제시:** 특정 기능 구현 방식에 대한 여러 아이디어나 대안을 요청하여 최적의 방법을 선택.

## 4. 개발 플로우 (Git 및 Ralph Mode 활용)

-   **브랜치 전략:** `main` 브랜치를 안정적인 버전으로 유지하고, 각 기능 개발은 `feature/<기능명>` 브랜치에서 진행합니다. (예: `feature/plan-agent-logic`, `feature/docker-setup`)
-   **커밋 메시지:** 의미 있는 커밋 메시지 (`feat:`, `fix:`, `docs:`, `refactor:` 등 Conventional Commits 지향)를 사용하여 변경 내용을 명확히 합니다.
-   **Ralph Mode (반복 개선):**
    -   `PROMPT.md` (이 문서)를 기반으로 AI 에이전트를 최소 2회 이상 반복 실행하여 코드 품질 및 문서화를 개선합니다.
    -   AI가 제안한 개선사항을 적용하고, 그 과정을 `docs/ralph-log.md`에 상세히 기록합니다.
    -   "어떤 프롬프트로 뭘 만들었고, 뭘 수정했는지"는 `ai-usage-log.md`에 기록합니다.

## 5. 소통 프로토콜 (나 자신과의 협업 및 AI와의 대화)

-   Git `Pull Request`를 활용하여 `feature` 브랜치에서 `main` 브랜치로 병합하기 전에 코드 변경 내용을 스스로 검토하는 습관을 들입니다.
-   AI 에이전트와의 모든 대화 및 프롬프트는 `ai-usage-log.md`에 기록하고, AI가 생성한 코드는 반드시 검증 과정을 거칩니다.
